\section{Open questions and next steps}\label{sec:open}

\begin{enumerate}
\item \textbf{Within-DP3 island stratification (the clean control).} Split the DP3 set by
epitope island count and size and recompute pass rates, to separate the
``contiguous-12 is easier'' confound from a real scaffolding-difficulty effect. This needs
the per-epitope island metadata in \texttt{datasets/dp2.parquet}.

\item \textbf{Move to a sigmoid transform (planned).} The composite currently maps each
metric to a percentile within its population, which is non-differentiable and
population-dependent, so it cannot be fit and a score does not transfer from one set to
another. A sigmoid (fixed midpoint and slope $k$ per metric) gives an absolute, saturating,
differentiable score, which both lets the weights be learned and lets a DP3-trained score
transfer to DP4. The scorer already supports it (\texttt{transform=sigmoid} in
\texttt{presets.py}); midpoints can be seeded from the DP3 separations (e.g.\ a cylinder
midpoint near 15, between the clash-free and clashing medians). This is the prerequisite for
fitting; the percentile version stays the default until then.

\item \textbf{Defer fitting until we have an experimental label.} The composite is a
single-layer perceptron with hand-set weights (feed-forward, no backprop), and the natural
next step is to learn the weights. We are deliberately \emph{not} fitting yet: the only label
available now is the in-silico \texttt{is\_pass}, which is defined by the four filters
themselves, so fitting to it would largely re-derive the filters. The plan instead is to
generate designs at scale, prioritize them with the current composite, run the top picks
through the PepSeq assay, and fit the weights against \emph{experimental binding}, the
non-circular target. The cylinder weight in particular is already justified by data: on DP3
it predicts the true antibody clash at AUC $0.93$ (Section~\ref{sec:noab}), so it earns
real weight even before a global fit.

\item \textbf{Confirm the canonical DP3 metrics builder.} Determine whether
\texttt{compute\_metrics\_mpnn.py} (currently archived) is the live RFD3+MPNN metrics
producer, and promote it into \texttt{episcaf\_analysis/} if so (see \texttt{docs/MIGRATION.md}).

\item \textbf{Validate the DP3 scoring path} end-to-end with \texttt{--preset antibody},
confirming \texttt{cylinder\_ca\_clashes} is the correct accessibility column there.

\item \textbf{Finish the 12-mer AlphaFold3 data migration} and verify landing counts
against the manifest.

\item \textbf{Pin down the crystal-gap dropouts.} We can already count them locally: 6M0J and
4WAT lose 15 and 40 tiles to unresolved crystal regions (Table~\ref{tab:tilecount}). What is
left is to confirm these against the cluster pre-filter status (the explicit
\texttt{crystal\_epi\_incomplete} entries) and to check whether the lost tiles fall in
contiguous runs, i.e.\ whether whole regions go un-scorable from gaps.
\end{enumerate}
